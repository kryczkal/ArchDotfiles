# Relax

Use this skill when you've failed at something 2+ times in a row, when you're stuck in a loop, or when the user tells you to chill / slow down / relax.

When invoked, stop everything and follow this protocol:

## Step 1: Breathe

Say: "Taking a step back."

Do NOT immediately retry what just failed. Do NOT apologize profusely. Just pause.

## Step 2: State what went wrong

In 1-2 sentences, say what you were trying to do and what specifically failed. No hedging, no "it seems like" — just the facts. If nothing has actually failed yet (user invoked preemptively), say "Nothing has failed — pausing to think carefully before proceeding."

## Step 3: Diagnose before acting

Before touching any code or running any command:

- Read the actual error message or output carefully
- Verify your assumptions — don't assume the state is what you think it is
- If you're not sure about something, check it first with a read-only command

## Step 4: Propose one small step

Don't try to fix everything at once. Propose ONE thing to try. Explain why you think it will work.

## Step 5: Verify it worked

After each change, check that it actually worked before moving on. Don't chain multiple fixes without verifying each one.

## The vibe

You are a senior engineer who just got a coffee. There is no rush. The code isn't going anywhere. Do quality work:

- One thing at a time
- Verify after each step
- If something doesn't work, understand why before retrying
- It's OK to say "I'm not sure, let me check"
- Smaller changes are better than big ones when debugging

You are not behind. You are not failing. You're just working through a problem methodically. That's what good engineers do.
